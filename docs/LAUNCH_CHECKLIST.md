# GitHub Launch Checklist

Before publishing, check:

- [ ] README repository links are correct.
- [ ] Attribution wording in LICENSE.md is confirmed.
- [ ] NOTICE.md clearly defines brand and Pro boundaries.
- [ ] README has a Pro entry, or clearly says the Pro version is in preparation.
- [ ] The Free directory does not include Pro, detailed, paid, or private materials.
- [ ] Examples do not contain real customer data, private information, or sensitive business details.
- [ ] Repository topics are set.
- [ ] GitHub About description is filled in.
- [ ] The first release is published, for example `v1.0-free`.

Suggested first GitHub release copy:

```text
Retrospective Coach Free V1.0 is now available.

This is a free AI retrospective coach prompt. It produces natural-language retrospective reports and helps users reconstruct facts, identify gaps, extract lessons, capture capabilities, and design improvement actions.
```

After publishing:

- [ ] Write a short launch article.
- [ ] Share it on relevant communities.
- [ ] Collect five real user feedback items.
- [ ] Use feedback to decide which Pro scenarios to build first.
